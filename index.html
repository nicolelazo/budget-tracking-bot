One-time server setup (only needed once, by whoever deploys)
Vercel → Settings → Environment Variables:

The two SUPABASE_* vars power both cross-device auto-save/restore and the Monday cron. The TELEGRAM_*/CRON_SECRET vars are only needed for the automatic weekly message.

  • SUPABASE_URL = your project URL, e.g. https://YOUR-PROJECT.supabase.co (Supabase → Settings → API → Project URL)
  • SUPABASE_SERVICE_KEY = your Supabase service_role key (Supabase → Settings → API)
  • TELEGRAM_BOT_TOKEN = token from @BotFather
  • TELEGRAM_CHAT_ID = your group chat ID
  • CRON_SECRET = any long random string (Vercel sends it to the cron)
  • APP_SYNC_SECRET (optional) = guards the sync + manual-run endpoints
Getting the chat ID: add the bot to the group, send any message, then open https://api.telegram.org/bot<TOKEN>/getUpdates and copy chat.id.

The Monday schedule lives in vercel.json (0 1 * * 1 UTC = 09:00 PH). Redeploy after adding the env vars.

This is exactly what will be sent to your group — nothing has been sent.

Email Submissions

Reads your Gmail (view-only) for the Google Form emails your chapters submit — liquidations and cash-advance requests — and logs them for you. It pulls the amount and details from the email, and automatically opens any receipt file or linked Google Sheet attached to the form so nothing is missed. Each one gets a short AI summary. Nothing is ever changed or sent from your inbox.

How to connect Google (one-time setup)
Google Cloud Console: enable Gmail API and Google Drive API → OAuth consent screen (add each team member's email as a test user) → create OAuth Client ID (Web application) and add every URL you open this app from as an authorized JavaScript origin — e.g. http://localhost:8917 and https://your-app.vercel.app.

This one connection powers both the email inbox here and private Google Drive receipt links in the Scan a Receipt tab.

Running locally? Open via start-dashboard.bat — Google sign-in does not work from a file:// page. On a deployed URL (Vercel) it works directly.
